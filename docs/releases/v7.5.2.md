# v7.5.2

## What changed

### Bug fix: `submit_phase_council_verdicts` tool registration

The `submit_phase_council_verdicts` tool was fully implemented, tested, and documented but never registered in the plugin manifest (`src/index.ts`). This made the tool invisible to the OpenCode runtime â€” the architect model's system prompt instructed it to call a tool that didn't exist in its toolset.

**Impact:** When `council_mode` was enabled in the QA gate profile, `phase_complete` would block with `PHASE_COUNCIL_REQUIRED` because the tool to write phase-council evidence was unreachable. The only workaround was manually writing the evidence file with the exact schema the gate validates.

**Fix:** Added the missing import and `tool: {}` block registration (2 lines in `src/index.ts`). Both existing guard tests (`plugin-tool-registration.test.ts` and `registration-smoke.test.ts`) flip from 3 failures to 7 passes.

### Test quality: mock cleanup pattern fixes

Fixed `mock.module` cleanup patterns across test files to prevent cross-test contamination in Bun's shared test runner process:

- Corrected `afterEach(mock.restore())` to `beforeEach(mockClear())` for top-level `mock.module()` patterns â€” `mock.restore()` removes module mocks permanently, breaking subsequent tests
- Fixed Node built-in spread violations â€” mock factories must spread real exports (`...realModule`) to avoid missing export errors
- Added `mockClear()` pattern to extracted mock function constants for deterministic test isolation
- Fixed import ordering (`afterEach` must be imported from `bun:test`)
- Updated CI anti-regression check to verify cleanup presence instead of banning cross-module mocks

## Why

The tool registration gap was discovered during Phase 10 of the mock contamination remediation work. The `phase_complete` gate blocked because the evidence-writing tool was unreachable. Root cause analysis traced to a registration gap in `src/index.ts` â€” the same bug class as commit `62faff3` which forgot `submit_council_verdicts`.

The mock cleanup fixes address cross-test contamination where `mock.module()` calls in one test file could leak into another via Bun's shared test runner process, causing non-deterministic failures.
