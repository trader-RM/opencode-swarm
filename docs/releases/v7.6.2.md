# v7.6.2

## What changed

### fix(gate): evidence-incomplete no longer hard-blocks task completion when session state confirms gates passed

`update_task_status("in_progress")` always creates `.swarm/evidence/<task>.json` with `gates: {}`. If evidence recording in the delegation hook subsequently fails silently (lock timeout, permission error), the gates stay empty on disk — causing `checkReviewerGate` to return `blocked: true` permanently, even though the in-memory session state was correctly advanced to `tests_run` by the same hook.

**Root cause:** `checkReviewerGate` treated an evidence file with incomplete gates as immediately authoritative, returning blocked without consulting session state, delegation chains, or plan.json.

**Fix:** `checkReviewerGate` now saves the specific missing-gates reason when evidence is incomplete and falls through to all existing session-state / delegation-chain fallback checks. Only after all fallbacks are exhausted does it block — using the evidence-specific reason for its more informative "missing gates: [X]" message.

Also replaces two occurrences of `/^\d+\.\d+$/` (which rejects N.M.P task ID format) with `isStrictTaskId()`, which correctly accepts three-segment task IDs.

**Files changed:**
- `src/tools/update-task-status.ts` — `checkReviewerGate`: save `evidenceIncompleteReason` and fall through to session-state / delegation-chain fallbacks instead of immediate block
- `src/hooks/delegation-gate.ts` — replace `/^\d+\.\d+$/` with `isStrictTaskId()` in two places for correct N.M.P task ID handling
- `src/tools/update-task-status.gates.test.ts` — three new regression tests: `gates:{} + tests_run → pass`, `gates:{} + idle → block (with gate names)`, `partial gates + tests_run → pass`

## Why

State advancement (in-memory) and evidence recording are independent operations in the `toolAfter` hook — state advances first and always; evidence write can fail silently. The previous fix aligned the incomplete-evidence path with the existing ENOENT path. This release completes the fix so that incomplete evidence no longer overrides a trustworthy in-memory session state.

## Migration steps

None. The fix is internal to the gate-checking logic.
