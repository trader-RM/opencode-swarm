# v7.15.0 — Lean Turbo Parallel Lane Execution

## What changed

### Lean Turbo — parallel lane execution strategy

Adds a new Turbo execution strategy (`strategy: 'lean'`) that organizes plan tasks into dependency-aware lanes and executes them concurrently. Each lane runs its own reviewer gate, with file-lock conflict detection to prevent parallel writers from stepping on each other.

**New modules** (`src/turbo/lean/`, 11 files):
- `planner.ts` — Wave-based topological sort (Kahn's algorithm) with lexicographic tie-breaking for deterministic lane assignment
- `runner.ts` — Parallel lane execution engine with configurable `max_parallel_coders` (default 2)
- `conflicts.ts` — File-lock conflict detection with TIER_3 security pattern matching (auth, crypto, permissions)
- `evidence.ts` — Per-lane evidence tracking and cross-lane aggregation
- `integration.ts` — Phase critic dispatch for holistic cross-lane review
- `phase-ready.ts` — Phase readiness verification with lane lock checks
- `reviewer.ts` — Per-lane reviewer gate dispatch
- `risk.ts` — Risk assessment for lane ordering and prioritization
- `state.ts` — Directory-scoped state management (no process-global flags)
- `task-completion.ts` — Session-aware task completion verification
- `index.ts` — Barrel exports for all 11 sub-modules

**New tools** (6 registered):
- `lean_turbo_plan_lanes` — Generate execution lanes from plan tasks
- `lean_turbo_acquire_locks` — Acquire file locks for a lane
- `lean_turbo_run_phase` — Execute a Lean Turbo phase
- `lean_turbo_review` — Dispatch per-lane reviewer gates
- `lean_turbo_status` — Query current Lean Turbo status
- `lean_turbo_runner_status` — Query runner state

**Configuration** (new in `opencode-swarm.json`):
```json
{
  "turbo": {
    "enabled": true,
    "strategy": "lean",
    "lean": {
      "max_parallel_coders": 2,
      "require_declared_scope": true,
      "conflict_policy": "degrade",
      "lane_review": true,
      "max_retries_per_lane": 2,
      "auto_acquire_locks": true,
      "evidence_per_lane": true,
      "fail_closed": true,
      "timeout_ms": 300000
    }
  }
}
```

### Bug fixes and hardening

- `hasActiveLeanTurbo()` now uses triple-AND guard: `turboMode === true && turboStrategy === 'lean' && leanTurboActive === true` — prevents false activation from partial config
- `stateUnreadable` changed from process-global `let` to directory-keyed `Map<string, boolean>` — fixes cross-session contamination (invariant #8)
- `.swarm` containment: all 6 Lean Turbo tool wrappers use `_directory` from tool context instead of `args.directory` (invariant #4)
- `DEFAULT_LEAN_TURBO_CONFIG` centralized as single source of truth in `src/config/constants.ts` — eliminates config drift between runner, planner, and status tools
- `verifyLeanTurboTaskCompletion` now checks `status === 'running'` and optional `sessionID` to prevent cross-session Stage-B bypass
- Auth pattern detection in `conflicts.ts` uses word-boundary regex to avoid false positives on `authentication.ts`
- `_internals` DI seam in `lean-turbo-run-phase.ts` — avoids `mock.module` contamination per AGENTS.md §7

### Modified existing files

- `src/parallel/file-locks.ts` — Lane-aware file locking with extended metadata
- `src/services/status-service.ts` — Lean Turbo status integration
- `src/commands/turbo.ts` — Lean Turbo command support
- `src/tools/phase-complete.ts` — Lean Turbo readiness gate
- `src/tools/update-task-status.ts` — SessionID-aware reviewer gate propagation
- `src/hooks/system-enhancer.ts` — LEAN_TURBO_BANNER injection for architect system prompt

## Why

Full-Auto mode executes tasks sequentially, which underutilizes available parallelism. Lean Turbo provides a safe parallel execution strategy that:
- Respects task dependencies via topological sort
- Prevents file-write conflicts via file-lock acquisition
- Maintains per-lane reviewer quality gates
- Composes with Full-Auto v2 (both can be active simultaneously)
- Fails closed — any state corruption or unreadable config defaults to safe serialized execution

## Migration steps

No migration needed — Lean Turbo is opt-in via config. Existing `strategy: "standard"` (or no strategy) behavior is unchanged.

To enable:
```json
{
  "turbo": { "enabled": true, "strategy": "lean" }
}
```

## Breaking changes

None.

## Known caveats

- `full-auto/state.ts` still uses a process-global `stateUnreadable` flag — this is benign in single-project scope but could cause issues if OpenCode ever runs multi-project single-process
- `TIER_3_PATTERNS` is duplicated between `task-completion.ts` and `update-task-status.ts` — advisory, not blocking
- `console.error` used in `lean-turbo-run-phase.ts` instead of project logger — advisory
