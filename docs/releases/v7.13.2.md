# v7.13.2

## Fixed

- **`scanDocIndex` event-loop hang in Bazel / Gradle / Cargo monorepos (issue #808)**: `scanDocIndex` was calling `fs.readdirSync(directory, {recursive: true})` which collects the entire directory tree synchronously before any pruning. In Bazel monorepos `bazel-out/` and `bazel-bin/` alone can contain 7 million+ entries, causing a 400+ second synchronous event-loop block on every cache-miss scan. The function is called from the `experimental.chat.system.transform` hook on every prompt, so this blocked every session in any Bazel workspace.

  **Fix**: Replaced the flat synchronous `readdirSync` call with an async recursive directory walk using `readdir({ withFileTypes: true })` from `node:fs/promises`. Skip directories (`bazel-out`, `bazel-bin`, `bazel-genfiles`, `bazel-testlogs`, `node_modules`, `dist`, `target`, `.gradle`, etc.) are now pruned **before descent** — their contents are never opened. `stat()` is called only for files that match doc patterns, not for every file in the repository.

- **Symlinked doc files were silently dropped**: The original async walk used `entry.isFile()` on `Dirent` objects, which returns `false` for symlinks. Symlinked documentation files (e.g. a `README.md` symlink in a `docs/` directory) were silently excluded from the index. Fixed by checking `entry.isSymbolicLink()` and following the link with `stat()` to determine the target type. Directory symlinks are intentionally **not** followed to avoid infinite cycles.

- **Root permission error silently poisoned the doc-manifest cache**: When the project root directory was unreadable, the async walk returned silently with no entries, and the code wrote an empty `doc-manifest.json` to disk. On subsequent calls this empty manifest passed cache validation (0 files, all mtimes valid), causing `scanDocIndex` to permanently return `cached: true` with 0 files — a silent correctness failure that persisted until the manifest was manually deleted. The fix tracks whether the root directory was successfully opened; if not, the function returns an empty manifest without writing it to disk, matching the old `readdirSync` behavior.

- **Native opencode agent sessions unnecessarily triggered `scanDocIndex`**: The `experimental.chat.system.transform` hook had no guard for native opencode agents (`build`, `plan`, `general`, `explore`, `compaction`, `title`, `summary`). These agents have no use for swarm context injection but still triggered `scanDocIndex` and the dark-matter scan on every prompt. Added an early-return guard that checks `OPENCODE_NATIVE_AGENTS` against the session's active agent and skips all swarm injection for native sessions.

## Additions to SKIP_DIRECTORIES

The following directories are now skipped before descent in `scanDocIndex`:
- `bazel-out`, `bazel-bin`, `bazel-genfiles`, `bazel-testlogs` — Bazel build outputs
- `target` — Maven (Java) and Cargo (Rust) build outputs
- `.gradle` — Gradle build cache

## Known caveats

- The native agent guard in `system-enhancer.ts` relies on `swarmState.activeAgent`, which is populated lazily by the `chat.message` hook. On the very first prompt of any new session the guard cannot fire because `activeAgent` has no entry yet. The async pruning walk ensures this first-prompt cost is bounded (proportional to matching doc files, not repo size), so the guard is an optimization rather than the primary safety mechanism.
